# Contributing to Awesome RAG

Thank you for your interest in contributing!

This repository aims to be a high-quality curated list of RAG (Retrieval-Augmented Generation) resources.

## Ways to Contribute

* Add new papers
* Add tools or frameworks
* Improve descriptions
* Fix broken links
* Suggest new sections

## Contribution Guidelines

### What to Add

* Relevant **RAG / Retrieval / LLM papers**
* Widely used or impactful tools
* High-quality resources

### What to Avoid

* Low-quality or unclear papers
* Irrelevant content
* Duplicate entries

## Format

Use this format:

```md id="fmt1"
- **[Title](LINK)** (Year)  
  > Short description
```

### Example

```md id="fmt2"
- **[Self-RAG](https://arxiv.org/abs/2310.11511)** (2023)  
  > Enables LLMs to retrieve, generate, and critique their own outputs.
```

## Structure

* Add items in the appropriate section
* Keep alphabetical or logical order
* Keep descriptions concise

## Process

1. Fork the repo
2. Create a branch (`add-paper-name`)
3. Commit your changes
4. Open a Pull Request

## Style

* Keep it clean and readable
* Use Markdown consistently
* Prefer arXiv links

Thanks for helping improve this project! ⭐
